# Rust crate 文档

## README

README 是人工 review 的入口。
它说明当前 crate 的职责、关键状态契约、公开 API 和验证命令。
它只陈述该 crate 自己拥有的模型和规则。

推荐结构：

```text
# crate-name
## 职责
## 状态契约
## 公开 API
## 设计
## 验证
```

职责使用具体动词，例如“定义”“校验”“转换”“返回事件”。
不要通过罗列别的 crate、层级或平台来反向定义边界。

## 设计文档

将维护决策放在 `docs/design.md` 或同等明确的相对路径。
设计文档可解释：

- 模型和模块按什么职责划分。
- 哪个聚合持有状态。
- 哪些不变量在构造或转换时校验。
- 命令、事件和错误如何表达。
- 为什么模块私有，以及根导出面如何保持稳定。

所有结论必须能在当前 crate 的代码或测试中找到证据。
未来规划、跨 crate 编排和实现历史不属于此文档。

## 验证

新增或修改文档后检查相对链接和 Markdown 结构。
文档描述行为时运行 `cargo test -p <crate>`。
文档引用公开 Rust API 时运行 `cargo doc -p <crate> --no-deps`。
