# 模式文档

## 意图

把跨项目工程经验写成独立、可搜索、可复用的 Markdown 模式文档。

## 适用场景

- 某个技术模式来自实际项目。
- 模式可以被其他项目复用。
- 内容偏解释和学习，不一定需要 agent 自动触发。

## 必须遵守的规则

- 每篇文档独立描述一个项目或一个模式。
- 标清来源项目。
- 写清适用场景和不适用场景。
- 必须包含反模式警示。
- 来源证据写在文末。
- 不引用 Arbor 私有上下文，除非模式本身只服务 Arbor。

## 推荐结构

- 一句话。
- 为什么需要这个模式。
- 核心架构或数据流。
- 关键设计。
- 反模式警示。
- 来源。

## 反模式

- 只写结论，不写证据。
- 把多个无关主题塞进一篇文档。
- 写成流水账，而不是可复用模式。
- 用没有上下文的代码片段替代规则。

## 证据

- `workspace/learn/patterns/vfs-pure-logic.md` 使用架构、关键设计、反模式和来源结构。
- `workspace/learn/patterns/rust-native-gui-dsl.md` 明确场景、边界和来源项目。
- `workspace/learn/patterns/README.md` 定义按来源项目和主题索引。
