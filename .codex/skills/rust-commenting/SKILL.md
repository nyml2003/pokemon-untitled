# Rust 注释规范

按 [rust-comments.md](references/rust-comments.md) 编写和审查注释。

## 默认流程

1. 先阅读目标 crate 的 `Cargo.toml`、公开 API 和现有注释。
2. 选择与目标代码相同的注释语言。
3. 按读者和范围选择 `//!`、`///` 或 `//`。
4. 只记录调用约束、行为、失败条件或实现原因。
5. 修改 rustdoc 后运行 `cargo doc -p <crate> --no-deps`。

## 硬规则

- 一个完整句子必须写在同一注释行；不要因行宽在句中换行。
- 不为名称、类型或直接控制流写重复说明。
- 函数用途、语义边界或返回条件不能从名称和签名直接判断时，必须补充简短注释；公开函数使用 `///`，私有函数使用 `//`。
- 不改变注释之外的行为，除非任务明确要求。
- 不要用注释替代类型、不变量、测试或错误建模。
