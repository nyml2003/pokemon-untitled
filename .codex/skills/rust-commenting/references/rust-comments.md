# Rust 注释规则

## 注释类型

- `//!` 用于 crate 或 module 的整体职责、边界和关键语义。
- `///` 用于公开类型、trait、枚举变体和函数的调用契约。
- `//` 用于私有实现中无法由代码直接看出的原因、约束或安全不变量。

## 内容

crate 或 module 文档说明它负责什么、不负责什么，以及调用方需要遵守的规则。
公开 API 文档说明输入含义、返回结果、失败条件、状态变化和副作用。
当函数用途、语义边界或返回条件不能从名称和签名直接判断时，补充简短注释说明它做什么；公开函数使用 `///`，私有函数使用 `//`。
实现注释说明为何采用该实现，或说明维护时容易破坏的约束。

不要解释显而易见的赋值、循环、字段访问或与名称重复的事实。
不要在注释中承诺代码和测试没有保证的行为。

## 换行与语言

每个完整句子独占一行。
不要为满足列宽在一句话的中间换行。
段落之间保留一个空的文档注释行。
代码块、列表和链接可以按 Markdown 结构换行。

保持目标文件的既有语言。
新 crate 或 module 没有既有语言时，优先使用与其相邻模块一致的语言。

## 验证

修改 `//!` 或 `///` 后，执行：

```sh
cargo doc -p <crate> --no-deps
```

注释涉及行为或示例时，同时运行该 crate 的测试。
