# 中文技术写作

按下面的规则写或改中文技术内容。

## 规则

- 先写结论，再写原因或动作。
- 优先用短句。一句话只放一个主意思。
- 主语、动作、条件、结果都要写清楚。
- 用常用词和具体动词。
- 术语只在有技术含义时保留。
- 删除不帮助理解、决策或执行的铺垫。
- 过程文档要写清下一步动作和完成标准。
- 代码注释解释行为、条件、风险或原因，不解释显而易见的语法。

## 避免

- 和你对齐
- 刚才确认
- 这里同步一下
- 拉齐认知
- 形成闭环
- 赋能
- 抓手
- 进一步地进行优化动作

同类会议腔、汇报腔、伪抽象词和空泛夸奖也要删掉。

## 改写流程

1. 抽出事实、动作、限制和结论。
2. 删除套话、重复和装饰性表达。
3. 按“结论 -> 原因 -> 步骤 -> 限制”重排。
4. 局部修补会保留混乱时，直接重写结构。
5. 复查事实没有被改坏，语气保持直接。
