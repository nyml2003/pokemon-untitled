# 原生构建与运行

> 分类：现状；最后核对：2026-07-20。
> 依据：`AGENT.md`、`ops-first-workflow` 与 `game-host` 的 Winit/WGPU 装配。

## 两个原生入口

`ops build game-host` 在 Windows 原生运行端构建 `game-host`。它验证目标平台可完成 native 构建，但不替代实际窗口、输入和 GPU 行为观察。

`ops run game-host` 会安全同步 Windows 镜像、在 Windows 端重建并启动渲染程序。它是原生窗口、Winit/WGPU surface、输入和实际帧提交的验收入口；不要在 WSL 直接启动产品入口。

## 运行前提与结果解释

原生验收只针对配置远端分支的当前提交。因此在调用前必须完成 WSL 验证、提交和推送，并确认镜像可同步。未提交或未推送的 WSL 修改不会被运行。

`BuildFailed` 和 `RunFailed` 表示原生阶段失败。保留 `ops` 输出的阶段事件和最终结构化错误，再修复对应的环境、镜像或提交问题；不要改用 Windows 原生命令绕过失败。

Windows 和 macOS 是 native 目标平台。一次 Windows 验收不自动证明 macOS 行为，反之亦然；涉及平台相关 Winit/WGPU 改动时，应在受影响目标上复核。
