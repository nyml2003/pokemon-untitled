# WSL 检查

> 分类：现状；最后核对：2026-07-20。
> 依据：`AGENT.md` 与 `ops-first-workflow` 的现行命令约束。

## WSL 的验证职责

WSL 是源码编辑、格式、静态检查和无窗口测试环境。常用命令如下：

| 目的 | 命令 |
| --- | --- |
| 格式检查 | `ops format --check` |
| Rust 静态检查 | `ops lint` |
| 全量单元测试 | `ops test` |
| 核心局部测试 | `ops test --suite core` |
| 世界局部测试 | `ops test --suite world` |

小范围改动可使用对应 suite；改变共享 crate、依赖图、状态合同或渲染提交边界时，应扩大到全量验证，或记录未执行的原因。命令结果和失败阶段由 `ops` 报告，不能以手动运行其他底层工具替代。

## WSL 不证明 native 行为

WSL 可以验证纯模型、用例、投影、帧计划和无窗口 adapter 合同。它不能证明 Windows/macOS 的窗口生命周期、GPU 驱动、surface 获取、输入法或真实用户交互。

不要在 WSL 直接启动 `game-host`。需要渲染验收时，先完成 WSL 检查、提交和推送，再通过 Windows 镜像执行 `ops build game-host` 或 `ops run game-host`。
