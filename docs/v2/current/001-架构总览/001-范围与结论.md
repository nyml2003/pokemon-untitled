# 当前架构

> 分类：现状；最后核对：2026-07-20
> 依据：根 `Cargo.toml`、`AGENT.md`、各 crate 的 `Cargo.toml`、公开根 API 和测试。

## 结论

项目是一个 Rust workspace。当前有 45 个 package。核心规则、用例、视图、外部能力和可执行程序分层组织。

```text
runtime -> adapter -> presentation -> application -> domain -> foundation
```

外层可以装配内层能力。内层不能反向依赖外层。`domain`、`application` 和 `presentation` 不依赖文件系统、窗口、GPU 或平台 API。

## 不变规则

- 领域和用例状态通过类型化命令、事件、快照和结构化错误流动。正常失败不依赖 panic。
- UI 和渲染不直接修改游戏、世界、战斗或编辑器状态。
- presentation 只描述画什么；adapter/runtime 才处理窗口、文件、GPU 和平台 API。
- 运行时是组合根，不新增游戏规则。
- `assets/` 与 `maps/` 属于 workspace。路径解析、目录扫描和文件读取留在 adapter/runtime。

## 文档边界

本文及所链接页面只记录已由代码、配置或验收结果证明的事实。Web runtime、持久活世界和未验收的 UI 能力不属于当前架构文档。

修改 workspace 成员、依赖方向、运行入口或权威状态所有者时，必须在同一变更中更新本组文档。单个 crate 的公开 API、状态不变量和内部设计取舍仍放在该 crate 的 README 或 `docs/design.md`。
