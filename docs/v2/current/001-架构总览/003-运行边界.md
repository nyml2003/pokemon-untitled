# 运行边界

> 分类：现状；最后核对：2026-07-20。
> 依据：根 `Cargo.toml`、各可执行 package 的 `Cargo.toml` 与 `src/main.rs`。

## 可执行入口

workspace 当前有四个 runtime 可执行程序和一个 adapter 中的数据生成工具。

| 程序 | package | 类型 | 直接效果 |
| --- | --- | --- | --- |
| `game-host` | `crates/runtime/game-host` | 产品运行时 | 创建 Winit 事件循环和原生窗口，装配游戏、地图、资产与 `NativeTarget`。 |
| `map-editor` | `crates/runtime/map-editor` | 创作工具 | 打开原生编辑器，加载项目与资产，保存编辑后的地图。 |
| `map-editor-cli` | `crates/runtime/map-editor-cli` | 命令行工具 | 检查、校验、打包、解包和语义 lint 地图项目。 |
| `tile-editor` | `crates/runtime/tile-editor` | 创作工具 | 打开原生瓦片语义编辑器。 |
| `game-data-import` | `crates/adapter/game-data-import` | 开发期生成工具 | 从指定 CSV 源目录生成并校验游戏数据 JSON。 |

前三个原生窗口程序的 `main` 都创建 `EventLoop` 并调用 `run_app`。`map-editor-cli` 的子命令是 `inspect`、`verify`、`pack`、`unpack` 和 `lint`。`game-data-import` 使用 `--source`、`--output` 和 `--version-group` 参数；它不是游戏产品的 runtime。

## runtime 是组合根

runtime 持有具体路径、窗口和外部实现。以 `game-host` 为例，它在启动时读取地图和叙事脚本，构造 `WorldApplication` 与 `GameSession`，加载资产，然后创建窗口与 `NativeTarget`。它还把 Winit 的键盘、文字和鼠标事件归一化，再交给表现状态或游戏命令。

因此 runtime 可以：

- 选择默认资源路径和命令行参数。
- 创建 Winit 事件循环、窗口和 native target。
- 把外部输入转为项目输入类型。
- 记录错误、请求重绘、调整 surface 和结束进程。

runtime 不可以：

- 在事件处理分支中复制战斗、世界、地图或编辑器规则。
- 让窗口或 GPU 对象进入 domain/application 的状态。
- 用渲染成功与否决定领域状态转换。

## 质量工具不进入产品链路

`quality/game-e2e` 依赖 `game-session`、`world-application` 和战斗 application，用于端到端游戏流程验证。`quality/wslg-wgpu-clear-smoke` 只依赖 Winit、WGPU 和 pollster，用于复现 WSLg/WGPU 环境问题。二者是 workspace 成员，但不是 `game-host` 的产品依赖。

Web runtime 不在当前入口集合中。现有 `future/web-runtime-webgl.md` 是未来文档，不能用来描述当前 native runtime 的行为。
