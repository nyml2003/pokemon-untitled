# 图示与校验

> 分类：现状；最后核对：2026-07-20。
> 依据：`docs/v2/CONVENTIONS.md`；已安装的 Mermaid CLI、PlantUML 与 Graphviz。

## 使用条件

图示只用于说明短文本难以完整表达的关系、时序或状态转移。它不能替代正文中的范围、结论和源码依据。一张图只回答一个问题，并紧挨对应的编号内容页。

| 工具 | 适用内容 |
| --- | --- |
| Mermaid | 分层、数据流、状态机和较短的运行流程。 |
| PlantUML | 参与者较多的时序图和组件关系图。 |

当前页面尚未引入架构图。以后加入时必须按下面规则把图示源码直接写入 Markdown。

## 图示资产

图示源码是页面中的 fenced code block。Mermaid 使用 `mermaid` 标记，PlantUML 使用 `plantuml` 标记。文档不保存图片、SVG 或独立图示源码；这样读者看到的文本就是可复制、可审阅、可校验的源内容。

图前写清范围；图后写出结论及对应 crate、类型或函数。没有证据定位的图不进入 `current/`。

## 必经校验

工具通过 Nix profile 持久安装：

```sh
nix profile add nixpkgs#mermaid-cli nixpkgs#plantuml nixpkgs#graphviz
```

每次新增或修改图示时，按以下顺序执行：

1. 运行 `ops docs check`。它从 Markdown 提取每个 Mermaid/PlantUML block 到系统临时目录并渲染为 PNG；提取或渲染失败时，命令失败。
2. 打开临时渲染结果，复核中文字体、箭头方向、边界、布局和文本可读性。
3. 在变更说明记录 `mmdc` 或 `plantuml` 版本，以及校验和视觉复核结果。

当前 profile 的工具版本为 Mermaid CLI `11.16.0`、PlantUML `1.2026.3`、Graphviz `14.1.2`。版本变化后，重新校验受影响的 Markdown 图示。
