# 图示与校验

> 分类：现状；最后核对：2026-07-20。
> 依据：`docs/v2/CONVENTIONS.md`；已安装的 Mermaid CLI、PlantUML 与 Graphviz。

## 使用条件

图示只用于说明短文本难以完整表达的关系、时序或状态转移。它不能替代正文中的范围、结论和源码依据。一张图只回答一个问题，并紧挨对应的编号内容页。

| 工具 | 适用内容 |
| --- | --- |
| Mermaid | 分层、数据流、状态机和较短的运行流程。 |
| PlantUML | 参与者较多的时序图和组件关系图。 |

当前页面尚未引入架构图。以后加入时必须按下面规则提交，而不是只提交 Markdown 代码块。

## 图示资产

图示源码和渲染产物与所属页面放在同一目录。Mermaid 源码使用 `.mmd`，PlantUML 源码使用 `.puml`，两者都生成并提交同名 `.svg`。正文引用 SVG，因此阅读结果不依赖托管平台是否支持 Markdown 图示语法。

文件名保留页面编号和主题，例如：

```text
004-场景投影与帧提交-运行流程.mmd
004-场景投影与帧提交-运行流程.svg
```

一个源码文件只生成一个 SVG。图前写清范围；图后写出结论及对应 crate、类型或函数。没有证据定位的图不进入 `current/`。

## 必经校验

工具通过 Nix profile 持久安装：

```sh
nix profile add nixpkgs#mermaid-cli nixpkgs#plantuml nixpkgs#graphviz
```

每次新增或修改图示时，按以下顺序执行：

1. 将源码渲染到仓库外的临时目录。Mermaid 使用 `mmdc -i <源码.mmd> -o <临时.svg>`；PlantUML 使用 `plantuml -tsvg -o <临时目录> <源码.puml>`。
2. 将临时 SVG 与准备提交的 SVG 比对。渲染失败或产物不一致时，不提交图示。
3. 打开临时渲染结果，复核中文字体、箭头方向、边界、布局和文本可读性。
4. 在变更说明记录 `mmdc` 或 `plantuml` 版本，以及比对和视觉复核结果。

当前 profile 的工具版本为 Mermaid CLI `11.16.0`、PlantUML `1.2026.3`、Graphviz `14.1.2`。版本变化后，重新渲染受影响的图示并完成同样的比对。
