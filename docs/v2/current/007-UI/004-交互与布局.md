# 交互与布局

> 分类：现状；最后核对：2026-07-20。
> 依据：`punctum-input`、`punctum-ui`、`game-ui` 与 `map-editor-view`。

## 输入与动作

平台输入先归一化为 `KeyEvent` 或 `TextEvent`。`PresentationState` 使用这些事件和快照产生表现更新或动作；地图编辑器则将键盘、滚轮、指针和 chrome 命中结果汇入 `EditorIntent`。UI tree 的 action 是类型参数，不以字符串回传给业务层。

已布局的 `UiFrame` 提供绘制命令、命中区域和 `UiActionHit<Action>`。页面交互应调用 `action_hit_at` 或 `hit_action` 取得类型化 action；低层 `hit_test` 仅返回 `UiId`，已被标记为 deprecated。

## 结构身份与布局

新页面使用 `UiNode::auto()`，由 `UiTree::new` 确定性分配结构 ID。动态节点使用 `UiKey` 保持稳定身份。手写 `UiId` 和 `UiNode::legacy` 已被标记为 deprecated，不应作为新页面接口。

布局由 `UiStyle`、尺寸、flex 方向、对齐、内边距、定位、边框和裁剪共同计算。`resolve` 输出的命令和命中区域位于同一坐标系，因此渲染与点击使用同一份布局结果。构建阶段和布局阶段分别返回 `UiBuildError`、`UiLayoutError`。

## 当前边界

通用焦点导航、滚动容器、文本测量/自动换行和可访问性语义树尚未作为通用能力验收。现有页面只能使用已经存在的局部行为；不能把这些目标写成当前 UI 合同。
