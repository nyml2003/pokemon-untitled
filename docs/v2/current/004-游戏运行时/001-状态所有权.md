# 游戏运行时

> 分类：现状；最后核对：2026-07-20
> 依据：`game-session`、`world-application`、`game-ui`、`game-scene-view`、`game-native-plan`、`game-native-target` 和 `game-host`。

## 状态所有权

| 状态 | 所有者 | 变更入口 | 外层读取形式 |
| --- | --- | --- | --- |
| 单局场景、世界、当前战斗和 roster seed | `GameSession` | `transition`、`advance_world_tick` | `GameSnapshot`、`GameEvents` |
| 世界地图规则、角色位置和朝向 | `World` | `transition`、`transition_actor` | `WorldOutcome` |
| NPC 脚本进度、角色外观和对白 | `WorldApplication` | 世界命令、`advance_npcs` | `WorldObservation` |
| 对战规则 | `Battle`，由 `BattleApplication` 持有 | 对战 action | 战斗观察、转换结果和回放步骤 |
| 菜单、焦点、动画、控制台和图鉴选择 | `PresentationState` | 归一化输入与 UI action | `PresentationSnapshot` |

`GameSession` 在世界与战斗场景之间切换。表现状态是独立的瞬态状态，不能直接修改 `GameSession`、`World` 或对战状态。

## 启动

```text
game-host
  -> game-fs-assets 读取资产和地图来源
  -> MapProject、WorldApplication、GameSession
  -> NativeAssets
  -> Winit 创建窗口
  -> NativeTarget 创建 WGPU runtime 与文本渲染器
```

`game-host` 持有窗口、native target、表现状态、资产与游戏会话。它负责加载失败的展示和进程退出；这些行为不进入 application 或 domain。

## 输入与逻辑 tick

```text
Winit 事件
  -> punctum-wgpu 输入归一化
  -> PresentationState
  -> PresentationAction / GameCommand
  -> GameSession::transition
  -> GameEvents + GameSnapshot
```

`PresentationState` 决定菜单、控制台、图鉴和动画的表现更新。需要改变游戏时，它返回动作，由 runtime 转成 `GameCommand` 并提交给 `GameSession`。

真实时间由 runtime 持有。`game-host` 只在世界场景活动时调用 `GameSession::advance_world_tick`；世界规则本身不读取时钟。

## 场景投影与帧提交

```text
GameSnapshot + PresentationSnapshot + MapProject + AtomicTileCatalog
  -> game-scene-view::project_scene
  -> SceneFrame
  -> game-native-plan::FramePlan
  -> game-native-target::NativeTarget::present
  -> punctum-wgpu::GpuRuntime
```

世界地图使用 Grid view。图鉴、战斗菜单和命令控制台使用 Pixel UI。`SceneFrame` 可以组合 Grid 基础画面与 UI overlay，也可以只包含 UI。所有路径最终产生 `FramePlan`。

场景投影和帧计划可以在没有窗口、GPU 或真实资产路径的测试中构造。GPU 或表面错误只影响当前运行时，不能反向改变游戏规则状态。

## 运行时失败

`game-host` 将启动、资源、场景投影、帧计划和提交失败输出到 runtime 边界。领域和用例层返回 `GameError`、`WorldError`、战斗错误或其他结构化错误，不写日志或退出进程。
