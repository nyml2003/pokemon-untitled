# 失败处理与恢复

> 分类：现状；最后核对：2026-07-20。
> 依据：`game-host/src/main.rs`、`game-scene-view`、`game-native-plan`、`game-native-target` 与 `punctum-wgpu`。

## 失败分类

| 阶段 | 错误形式 | 当前处理者 |
| --- | --- | --- |
| 启动 | 地图、脚本、数据、资产、窗口或 native target 构造失败。 | `main`/`CreatureGameApp::new` 返回启动错误。 |
| 游戏转换 | `GameError`、世界或战斗错误。 | runtime 决定显示或记录；状态转换通过 `Result` 返回。 |
| 场景投影 | `SceneViewError`。 | `redraw` 输出错误并退出事件循环。 |
| 帧计划 | `FramePlanError`。 | `redraw` 输出错误并退出事件循环。 |
| Native 提交 | `NativeTargetError` 或 `PresentOutcome`。 | runtime 按 outcome 恢复、跳过或退出。 |

内层只返回可判断的结构化错误。只有 runtime 拥有 `eprintln!`、窗口重绘和 `event_loop.exit()` 等进程级副作用。

## 可恢复的 surface 结果

`PresentOutcome` 区分已提交、已提交但需重配、最小化跳过、超时跳过、遮挡跳过、仅重配和 surface 丢失。`game-host` 对 `Reconfigured` 与 `SurfaceLost` 调用 `resize` 并请求重绘；其余成功或跳过结果不改变游戏状态。

窗口 resize 也只调整 `NativeTarget` 的 surface 尺寸并请求重绘。最小化、超时和遮挡意味着当前帧没有提交，不意味着命令失败或世界状态回滚。

## 不可恢复的当前处理

场景投影、帧计划和 `NativeTarget::present` 返回 `Err` 时，当前 `game-host` 输出错误并退出事件循环。它不尝试从 GPU 错误重建游戏 session，也不把提交失败写成游戏事件。下一次进程启动从正常装配路径重新开始。

这条边界防止外部错误污染 `GameSession`：游戏规则的失败由命令转换表达，渲染基础设施的失败由运行时生命周期表达。
