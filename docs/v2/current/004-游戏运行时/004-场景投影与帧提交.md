# 场景投影与帧提交

> 分类：现状；最后核对：2026-07-20。
> 依据：`game-host::redraw`、`game-scene-view::project_scene`、`game-native-plan::FramePlan` 与 `game-native-target::NativeTarget`。

## 场景投影

每次重绘时，`game-host` 先推进表现状态，依据 surface 尺寸计算 `game_viewport`，再取得 `GameSnapshot` 和 `PresentationSnapshot`。它把这些值、地图项目、`AtomicTileCatalog`、图鉴数据和可选控制台交给 `project_scene`。

`project_scene` 选择以下 `SceneFrame`：

| 条件 | 基础画面 | 可选覆盖层 |
| --- | --- | --- |
| 世界场景 | 地图与角色组成的 `GameView`。 | 控制台 `UiFrame`。 |
| 战斗场景 | 战斗 Pixel UI `UiFrame`。 | 控制台 `UiFrame`。 |
| 图鉴开启 | 图鉴 `UiFrame<PokedexAction>`。 | 控制台 `UiFrame`。 |

因此结果可以是单独 Grid、单独 UI，或 `GridWithUi`、`UiWithUi`、`PokedexWithUi`。投影函数只返回 `ProjectedScene`；地图投影、UI 构建或布局失败通过 `SceneViewError` 返回。

## 从视图到帧计划

`game-host` 对每一种 `SceneFrame` 调用 `FramePlan::from_game_view` 或 `FramePlan::from_ui_frame`。带覆盖层的场景先生成基础 pass，再生成 overlay pass，最后用 `FramePlan::compose` 组合。`NativeAssets` 在此之前已经把解码图像构造成 atlas 和资源 ID。

构建计划会校验必需 surface、白色 UI 资源、资产键、尺寸和 GPU 计划。失败以 `FramePlanError` 返回，尚未调用 WGPU。`FramePlan` 包含 pass、GPU `SubmissionPlan`、文本标签、viewport 和文本缩放，是 presentation 到 adapter 的唯一 native 提交数据。

## Native 提交

`NativeTarget::present` 读取每个 frame pass 的 GPU plan，交给 `GpuRuntime::present_plans_with_overlays`，并在每个 pass 中编码 glyphon 文本。它持有 WGPU runtime 和文本 GPU 资源；presentation 不依赖它们。

提交结果为 `PresentOutcome` 或 `NativeTargetError`。因此场景投影与帧计划可以在没有窗口/GPU 的单元测试中验证，真实 surface 的获取、重配和提交留给 adapter/runtime。
