## 资产与数据

资产和地图是 workspace 资源。`game-fs-assets` 读取本地文件，并按 `AssetRequest`、`AssetCatalog` 或地图图块来源返回数据。presentation 使用 `AssetKey`、解码图像、图块目录和 `FramePlan`，不持有 `PathBuf`。

`game-data` 保存可嵌入、只读的数据集。`game-data-import-core` 解析并校验 PokeAPI CSV；`game-data-import` 才读取目录和写出生成结果。

## 错误模型

| 边界 | 失败表达 |
| --- | --- |
| foundation/domain/application | `Result`、诊断或明确错误枚举。 |
| presentation | 视图、布局、资源键或帧计划错误。 |
| adapter | 平台、格式、文件、GPU 或第三方库错误，转换为项目错误。 |
| runtime | 格式化错误、日志、窗口关闭和进程退出。 |

用户输入、资产缺失、状态冲突和格式错误不是 panic 条件。`expect` 只用于构造器或本地不变量已经保证的条件。

## 平台与开发

WSL 是编辑、格式检查、静态检查和无窗口测试环境。Windows Git 镜像只接收已提交并推送到配置远端分支的内容。原生构建和试玩使用 Windows 或 macOS 目标平台。

`ops` 是项目的开发与验证入口。完整命令、镜像规则和 native 验收方式见[当前开发与原生验收](../../008-开发与验收/README.md)。

## 质量验证

| 范围 | 当前验证 |
| --- | --- |
| 纯模型、规则和 reducer | 对应 crate 的单元测试。 |
| 场景和帧计划 | 纯投影与编码契约测试，不依赖真实窗口作为唯一证据。 |
| 可玩流程 | `game-e2e` 验证世界进入战斗、战斗完成和返回世界。 |
| 图形环境 | `wslg-wgpu-clear-smoke` 复现 WSLg/WGPU 环境问题。 |
| 原生窗口与 WGPU 行为 | 在目标桌面平台通过 `ops build game-host` 或 `ops run game-host` 验收。 |

`quality` crate 不作为产品 runtime 的依赖。
