# 平台与开发

> 分类：现状；最后核对：2026-07-20。
> 依据：`AGENT.md`、`008-开发与验收/` 与 runtime/adapter 的平台依赖。

## 环境分工

WSL 用于编辑、格式检查、静态检查和无窗口测试。Windows 与 macOS 是 `game-host` 的原生构建、运行和窗口/GPU 验收平台。WSLg 冒烟只帮助定位图形环境问题，不能证明桌面产品可玩。

Windows Git 镜像只接受已经提交并推送到配置远端分支的内容；它不会读取 WSL 未提交修改，也不自动 stash、clean、merge 或 reset。镜像状态异常时应按 `ops` 的结构化诊断处理，不手工覆盖镜像目录。

## 统一操作入口

仓库把格式、检查、测试、同步、原生构建和运行收敛为 `ops` 工作流。常用入口及其顺序见 [008 开发与验收](../008-开发与验收/README.md)。文档和自动化不应另行发明直接调用 Cargo、PowerShell、cmd 或 Windows 原生命令的并行流程。

涉及 Winit、WGPU、窗口或 surface 的改动，WSL 至少做无窗口检查；最终行为仍要在目标桌面平台运行验证。

## 平台类型不得进入核心

Winit/WGPU 类型、文件路径、终端句柄和浏览器 API 位于 adapter/runtime。domain、application 和 presentation 只通过输入、快照、帧计划、字节和结构化结果连接外部世界。Web 是未来独立 runtime/adapter，不应通过 `cfg` 分支把浏览器 API 扩散进当前核心。
