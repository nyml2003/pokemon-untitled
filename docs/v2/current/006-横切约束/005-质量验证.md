# 质量验证

> 分类：现状；最后核对：2026-07-20。
> 依据：各 crate 测试、`quality/game-e2e`、`quality/wslg-wgpu-clear-smoke` 与 `AGENT.md`。

## 验证分层

| 范围 | 首选证据 | 不足以证明的内容 |
| --- | --- | --- |
| foundation/domain | 构造、规则、错误和边界的单元测试。 | 原生窗口或真实 GPU 行为。 |
| application | 命令、事件、快照、reducer 与历史测试。 | 平台输入和文件 I/O。 |
| presentation | 投影、UI 布局、资源键与 `FramePlan` 契约测试。 | surface 获取和驱动兼容性。 |
| adapter | 格式、I/O 转换、输入归一化和提交结果测试。 | 产品完整流程。 |
| runtime/native | 目标平台实际构建和运行。 | 仅 WSL 的编译成功。 |

测试应覆盖正常失败路径。对于场景和提交计划，纯输入/输出测试比窗口截图更稳定；截图或手工试玩不能替代状态和计划契约测试。

## 当前质量 crate

`game-e2e` 以 `game-session`、`world-application` 和对战 application 构造可玩流程，验证从世界进入战斗、完成战斗并返回世界等跨用例行为。它不依赖产品窗口。

`wslg-wgpu-clear-smoke` 只创建 Winit/WGPU 最小链路，用于重现或确认 WSLg/WGPU 环境。它不加载游戏、地图、UI 或资产，因此不是 `game-host` 的原生试玩验收。

## 执行边界

按改动范围通过 `ops` 运行格式、检查和测试。共享依赖图或渲染提交边界变更需要扩大到 workspace 级验证，或在变更记录中说明未运行原因。原生窗口和 GPU 行为必须在 Windows/macOS 目标平台通过 `ops build game-host` 或 `ops run game-host` 复核。
