# 平台边界

> 分类：现状；最后核对：2026-07-20。
> 依据：`AGENT.md` 与 `008-开发与验收/` 中的现行工作流说明。

## 平台职责

当前图形产品入口是原生 `game-host`。Windows 和 macOS 是原生构建、运行和验收平台；它们使用 Winit/WGPU native 链路。WSL 是编辑、格式化、静态检查和无窗口测试环境，不能替代桌面试玩验收。

| 环境 | 可以做的事 | 不能代替的事 |
| --- | --- | --- |
| WSL | 编辑文档和代码、运行格式检查、静态检查、单元测试与无窗口测试。 | Windows/macOS 的窗口、GPU 驱动和用户交互验收。 |
| Windows | 通过 Windows Git 镜像构建和运行 `game-host`。 | 读取 WSL 中未提交、未推送的工作树内容。 |
| macOS | 原生构建、运行和验证 `game-host`。 | 为其他平台的窗口后端背书。 |

涉及 Winit、WGPU、窗口尺寸或 surface 行为的改动，必须在目标桌面平台实际运行。WSL 至少执行可用的编译和无窗口验证；`wslg-wgpu-clear-smoke` 只用于定位 WSLg/WGPU 环境问题，不是产品验收替代品。

## 平台 API 的位置

Winit、WGPU、窗口句柄、文件路径和终端 I/O 位于 runtime 或 adapter。`punctum-wgpu` 提供 Winit 输入归一化与 GPU runtime，`game-native-target` 完成 native 帧和文本提交。domain、application 与 presentation 不依赖这些平台 API。

这个边界允许核心状态、场景投影和 `FramePlan` 在无窗口环境中被构造和测试。它不承诺任何未实现的平台后端。

## 当前不包含的目标

浏览器 API 和 Web runtime 不属于当前 core 或 native runtime。Web 方向只在 `docs/v2/future/` 记录；它不能促使当前 domain/application/presentation 引入浏览器类型，也不能改变现有 Windows、macOS 和 WSL 的验收边界。
