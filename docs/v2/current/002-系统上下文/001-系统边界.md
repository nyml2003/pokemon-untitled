# 系统上下文

> 分类：现状；最后核对：2026-07-20
> 依据：根 `Cargo.toml`、`AGENT.md`、runtime crate 和资产目录。

## 系统边界

Pokemon Untitled 当前提供单机游戏、原生地图编辑器、地图命令行工具、瓦片语义编辑器和游戏数据导入工具。它们共享同一个 Rust workspace，但各自拥有运行时入口。

```text
玩家 ──────────────> game-host ──────────────> 原生窗口与 GPU
地图作者 ───────────> map-editor ────────────> 原生窗口与 GPU
自动化或地图作者 ───> map-editor-cli ───────> 本地项目文件
内容作者 ───────────> tile-editor ───────────> 瓦片语义 JSON
数据维护者 ─────────> game-data-import ──────> 生成的游戏数据

所有入口 <────────── assets/、maps/ 与内嵌 game-data
```

`assets/` 与 `maps/` 是 workspace 资源，不属于任何单独 crate。`game-data` 保存内嵌的只读游戏数据。运行时路径、目录扫描和文件读取由 runtime 或 adapter 负责。

## 当前运行入口

| 入口 | 位置 | 使用者 | 输出或效果 |
| --- | --- | --- | --- |
| `game-host` | `crates/runtime/game-host` | 玩家 | 原生桌面游戏。 |
| `map-editor` | `crates/runtime/map-editor` | 地图作者 | 原生地图编辑器。 |
| `map-editor-cli` | `crates/runtime/map-editor-cli` | 自动化、地图作者 | 交互式命令、检查、校验、打包和解包。 |
| `tile-editor` | `crates/runtime/tile-editor` | 内容作者 | 原生瓦片语义编辑器。 |
| `game-data-import` | `crates/adapter/game-data-import` | 数据维护者 | 从 PokeAPI CSV 生成游戏数据。 |

`game-data-import` 是 adapter 中的可执行工具，不是产品游戏 runtime。

## 外部边界

| 外部能力 | 当前实现 | 内层看到的形式 |
| --- | --- | --- |
| 本地资产读取 | `game-fs-assets` | 资产目录、资产字节和已解码图像。 |
| 地图二进制容器 | `map-project-storage` | 已校验的 `MapProject`、字节流和结构化错误。 |
| 窗口、键盘、指针和 GPU | `punctum-wgpu`、`game-native-target` | 已归一化输入、`FramePlan` 和 `PresentOutcome`。 |
| 原生文本渲染 | `game-native-target` 与 glyphon | `NativeTextLabel`。 |
| 终端 | `punctum-crossterm` | 平台无关输入和 terminal plan。 |
| Ramus 命令能力 | `battle-ramus-adapter` | 受授权的对战 action。 |

平台或库错误必须在 adapter/runtime 层转换。它们不能作为平台错误码进入 domain 或 application。

## 平台

Windows 和 macOS 是 `game-host` 的原生构建和试玩平台。WSL 用于编辑、格式检查、静态检查和无窗口测试，不作为桌面试玩环境。

浏览器 API 不属于当前 core 分层或原生 runtime。当前仓库没有 Web runtime。
