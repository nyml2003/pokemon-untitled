# 命令行工具

> 分类：现状；最后核对：2026-07-20。
> 依据：`crates/runtime/map-editor-cli/src/main.rs` 与 `src/lib.rs`。

## 两种运行模式

`map-editor-cli` 在第一个参数是 `inspect`、`verify`、`pack`、`unpack` 或 `lint` 时进入子命令模式；其他情况进入交互式模式。两种模式都在 runtime 中读取路径、加载资产 ID 与语义目录，再调用核心模型或 storage crate。

交互式模式从标准输入读取 JSON Lines `EditorVirtualCommand`，交给库函数 `execute`，把 `CliResponse` 写回标准输出。`SaveRequested` 由 runtime 的 `save_project` 执行，按目标扩展名写 JSON 或 `.g3mp`。协议模型与编辑逻辑不读取标准输入或文件路径。

## 文件子命令

| 子命令 | 输入 | 行为 |
| --- | --- | --- |
| `inspect` | 一个 `.g3mp` 文件。 | 使用 `MapProjectReader::inspect` 输出容器元数据。 |
| `verify` | 一个 `.g3mp` 文件。 | 读取并验证容器和地图。 |
| `pack` | JSON 输入和 `.g3mp` 输出。 | 读取 JSON 项目，使用 `MapProjectWriter` 写二进制容器。 |
| `unpack` | `.g3mp` 输入和 JSON 输出。 | 读取容器，使用 `to_json_pretty` 写 JSON。 |
| `lint` | 一个地图或目录，可选 `--json`。 | 读取项目并输出瓦片语义诊断。 |

`lint` 在目录中只处理扩展名为 `.json` 或 `.g3mp` 的地图文件。出现诊断时，命令返回失败；`--json` 只改变报告形式，不跳过语义检查。

## 责任边界

CLI 决定参数语法、标准输入输出、退出错误、默认项目路径和 `std::fs` 操作。`map-project-storage` 只接收字节或 `Read`/`Write`；`map-editor-core` 只处理模型和命令；`map-tile-semantics` 只产生诊断。这样自动化调用和原生编辑器可共享同一校验与状态规则。
