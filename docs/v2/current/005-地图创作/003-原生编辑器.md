# 原生编辑器

> 分类：现状；最后核对：2026-07-20。
> 依据：`crates/runtime/map-editor/src/main.rs`、`assets.rs`、`map-editor-view` 与 `game-native-target`。

## 加载与装配

`MapEditorApp::new` 先加载 workspace `assets/` 中的图块资源、资产目录和瓦片语义，再从第一个命令行参数或默认路径选择项目。`load_project` 按文件扩展名读取 `.g3mp` 或 JSON；随后 `EditorModel::with_semantics` 接收项目、图块 ID 和语义目录。

窗口在 `initialize` 中创建，初始逻辑尺寸为 1600x950。runtime 用已加载的 `NativeAssets` 创建 `NativeTarget`，保留 `EditorModel`、`EditorController`、原子图块目录、语义目录、viewport、光标和窗口对象。

## 工作台到 native 帧

每次重绘时，runtime 将 `EditorModel`、目录与 viewport 交给 `map-editor-view::project`。该 crate 产出地图内容和编辑器 chrome 的视图结果。runtime 将地图和 chrome 分别转换为 `FramePlan`，并在需要时组合 pass，最后调用 `NativeTarget::present`。

Winit 输入先变为 core 的键盘、滚轮、指针意图，随后通过 controller 或 `intent_for_ui_action` 进入 `EditorModel::reduce`。展示层不保存项目，也不处理磁盘错误。

## 保存与语义诊断

收到 `SaveRequested` 后，runtime 先运行 `TileSemanticsCatalog::lint`，再将 `MapProject` 用 `to_json_pretty` 序列化并写入当前项目路径。原生编辑器的保存路径当前写 JSON；`.g3mp` 写入由 CLI 的 pack 或扩展名分支处理。

加载、校验、序列化、文件写入、窗口或提交错误由 runtime 保存为编辑器错误状态或输出到进程边界。`EditorModel` 只接收错误文本和已保存标记，不直接拥有文件系统。
