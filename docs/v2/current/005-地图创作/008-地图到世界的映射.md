# 地图到世界的映射

> 分类：现状；最后核对：2026-07-20。
> 依据：`world-application::from_map_project`、`from_map_project_with_scripts`、`with_npc_scripts` 与 `game-host` 地图装配。

## 映射不是文件读取

地图项目是创作模型；`WorldApplication` 是可运行的世界用例。二者之间的映射发生在 application 边界，避免 world-domain 依赖地图 JSON、图块图像或编辑器状态。

`from_map_project` 逐格读取 `collision_cells` 与 `event_cells`，生成 `world_domain::Tile`：

| 地图 cell | 世界 tile |
| --- | --- |
| `Blocked` | `Wall` |
| `Walkable` + `Encounter` | `Grass` |
| `Walkable` + 无事件 | `Ground` |

视觉素材不参与此规则。更换地图图块图片不会改变可通行性或遭遇语义；相反，编辑 collision/event 会改变可运行世界。

## 角色和外观的分离

项目的 `MapActor` 映射为 `WorldActor`，包含类型安全 ID、位置、朝向和可移动标记。外观不放进 world-domain actor，而由 `WorldApplication` 的 `appearances: BTreeMap<WorldActorId, CharacterAppearanceId>` 保存。玩家使用固定玩家 ID 和默认红色外观；地图 NPC 使用项目中的外观。

`observe` 在构造 `WorldObservation` 时要求每个 world actor 都有外观，缺失时返回 `WorldApplicationError::MissingAppearance`。这阻止损坏项目在渲染时悄悄漏掉角色。

## 脚本绑定的防线

`from_map_project_with_scripts` 先完成地图映射，再调用 `with_npc_scripts`。每个 `ScriptProgram` 必须：有绑定 actor、不能控制 `actor:player`、actor 名符合前缀格式、目标 actor 存在，且同一 actor 只能绑定一个脚本。

这些检查分别对应 `UnboundScript`、`ScriptControlsPlayer`、`MalformedScriptActor`、`ScriptActorMissing` 和 `DuplicateScriptActor`。脚本不是 runtime 根据名称猜测附着的文件；绑定在 application 构造阶段显式失败。

## 给运行时的结果

`game-host` 负责从默认地图和内嵌叙事脚本取得输入，再调用 `WorldApplication::from_map_project_with_scripts`。成功后它只持有已构造的 `WorldApplication` 和用于渲染的原始地图项目；世界命令、NPC 推进和观察都经 application API。

新增地图语义时，需要决定它是创作/运行映射的一部分还是纯视觉数据。前者应在此映射中形成明确 world 规则并有测试；后者不应进入 `world-domain`。
