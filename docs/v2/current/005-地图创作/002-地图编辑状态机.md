# 地图编辑状态机

> 分类：现状；最后核对：2026-07-20。
> 依据：`map-editor-core` 的 `EditorModel`、`EditorIntent`、`EditorController`、输入模块与 `map-editor-cli` 库入口。

## 编辑模型与 reducer

`EditorModel` 持有当前 `MapProject`、原子图块、选择、工具、历史、视图范围、帮助状态和错误文本。编辑操作不在窗口事件中直接改写项目，而是经 `reduce` 处理 `EditorIntent`：

```text
输入或 UI action
  -> EditorIntent
  -> EditorModel::reduce
  -> 下一个 EditorModel + EditorEffect
  -> runtime 处理 effect
```

意图覆盖选择图块或素材、切换工具、绘制、添加/删除图层或素材、撤销、重做、保存和帮助开关。`Save` 返回 `EditorEffect::SaveRequested`，而不是在 core 中写磁盘。正常编辑返回新 model，因此历史和失败路径可在无窗口测试中验证。

## 控制器与输入路由

`key_intent` 将已归一化键盘事件映射为保存、撤销、重做、工具选择、图块选择和帮助等意图。`wheel_intent` 移动图块选择。`EditorController` 把指针位置、按键与 map viewport 转换为画笔、选择或视图意图；`map-editor-view` 把命中的 chrome action 转为同一套 `EditorIntent`。

因此键盘、滚轮、鼠标和 UI 点击在 reducer 之前汇合。布局和命中测试属于 presentation，实际编辑规则、历史和效果属于 application。

## virtual command 边界

`EditorVirtualCommand` 为非窗口调用提供同一模型的 JSON 命令表示。`map-editor-cli` 的库入口 `execute` 接收 `EditorModel` 和 virtual command，返回更新后的 model 与 `CliResponse`；`save_requested` 再判断响应中是否出现 `SaveRequested`。

CLI 负责逐行读取 JSON、序列化响应、选择文件和写入项目。virtual command 不能绕过 reducer，也不持有 `PathBuf` 或文件句柄。
