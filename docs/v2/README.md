# 文档索引（v2）

> 分类：现状；最后核对：2026-07-20

本目录只保存会随项目继续维护的文档。文档按事实状态，而不是按作者、日期或功能领域分类。

| 分类 | 位置 | 含义 |
| --- | --- | --- |
| 现状 | [`current/`](current/) | 已由当前代码、配置或已验收运行结果证明的事实。 |
| 未来 | [`future/`](future/) | 尚未完成的目标、提案和待决策事项。不能把它当成实现依据。 |
| 过时 | 不保留文件 | 已被代码或新决策替代的文档直接删除。需要背景时查 Git 历史。 |

## 权威顺序

现状问题以代码和配置为准：根 `Cargo.toml`、目标 crate 的 `Cargo.toml` 与 `src/`、`AGENT.md`，以及已通过的验收结果。crate 的 README 和 `docs/design.md` 紧贴其代码，仍是该 crate 的现状文档；数据来源和许可证文件继续随资产保存，不移入本目录。

文档与实现冲突时，先修正文档或代码，再继续引用。不要用未来文档推断当前已经存在的 API、运行时或行为。
