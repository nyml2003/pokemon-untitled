# Gen3 单打战斗系统：当前状态与路线图

## 结论

当前引擎已经支持可回放的六对六 Gen3 风格单打基础战斗、主要异常状态、能力等级、部分
变化招式、四种天气以及一批特性。它还不是完整的《宝可梦 绿宝石》单打规则实现。

最主要的缺口不在基础伤害，而在易失状态、蓄力/多回合招式、场地状态、特殊伤害规则、招式
效果资料映射覆盖率，以及尚未实现的特性触发时机。

本文件描述当前代码的实际能力，不把历史开发阶段等同于完成状态。

## 范围与非目标

- 目标：逐步对齐《宝可梦 绿宝石》的 Gen3 六对六单打规则，并保持确定性回放和可观察事件。
- 当前不支持：道具、双打、联机同步、跨世代规则、捕获及战斗外背包系统。
- 道具不会以空枚举、占位字段或无效行动预埋到领域模型；未来应作为独立阶段实现。

## 当前已实现

### 基础战斗

- 六对六单打、同时提交命令、优先度和速度排序、主动换人、濒死强制换人、逃跑、胜负和残局平局。
- PP、PP 耗尽后的挣扎及其反冲。
- 属性相克、STAB、暴击、命中/闪避等级、85--100 伤害随机数，以及固定种子回放。
- 招式自身的 `MoveCategory` 决定物理/特殊结算，不再按属性重新推导。
- 领域事件会投影到观战、会话、播放和中文战斗消息。

### 状态、能力等级与已映射招式效果

- 主要异常：烧伤、普通中毒、剧毒、麻痹、睡眠、冰冻；包含免疫、行动前阻止、回合末伤害、
  剧毒递增、烧伤物攻修正和麻痹速度修正。
- 伤害后概率附加主要异常，包括资料驱动的效果概率，例如 10% 烧伤。
- 七项能力等级（攻击、防御、特攻、特防、速度、命中、闪避）的 -6 至 +6 变化、换人清空、
  伤害/速度/命中读取，以及暴击与能力等级的 Gen3 交互。
- 已映射的变化招式效果：自身或目标能力变化、固定比例回复、守住、替身、黑雾、自我暗示、
  睡觉、自我刷新和部分伤害后能力下降。
- 畏缩、吸取、反冲，以及音爆、龙之怒、地球上投、黑夜魔影等确定性固定伤害。

### 天气

- 晴天、下雨、沙暴、冰雹；普通天气招式的五回合时长、天气覆盖、永久天气入场特性和回合末事件。
- 沙暴/冰雹回合末伤害及其属性免疫。
- 晴雨对火/水招式伤害的修正；雷电的晴雨命中规则；气象球的天气属性、威力和 Gen3 物特分类。
- 叶绿素、悠游自如、沙隐、雨盘，以及无关天气/气闸对天气效果的抑制。
- 未实现：太阳光束蓄力、场地状态和其余天气专属招式交互。

### 已实现特性

| 类别 | 特性 |
| --- | --- |
| 主要异常免疫 | 免疫、柔软、水幕、不眠、干劲、熔岩铠甲 |
| 属性无效/吸收 | 飘浮、蓄水、蓄电、闪火 |
| 入场与换人 | 威吓、降雨、日照、扬沙、自然回复 |
| 能力等级防护 | 清晰之躯、白色烟雾、怪力钳、锐利目光 |
| 状态/回合末 | 蜕皮、早起、同步、加速、神奇鳞片、根性 |
| 暴击/附加效果 | 战斗盔甲、硬壳盔甲、鳞粉、天恩、精神力 |
| 伤害、命中与数值 | 坚硬脑袋、压迫感、猛火、茂盛、激流、虫之预感、大力士、瑜伽之力、厚脂肪、复眼、活力 |
| 天气与限制 | 叶绿素、悠游自如、沙隐、雨盘、无关天气、气闸、踩影、沙穴 |

闪火会无效火属性伤害与变化招式；激活后仅在本次上场期间使自身火属性伤害招式威力提高至
1.5 倍。污泥浆会将命中非替身目标后的吸取回复改为对攻击方造成同量特性伤害。

### 资料与可观察性

- PokeAPI 快照包含招式基础资料、`effect_id`、`effect_chance`、可学习招式和 Gen3 及以前的
  宝可梦特性资料及中文名称。
- 组队默认选择第一个非隐藏常规特性；未映射特性仍可作为资料存在，但不会伪造规则效果。
- 当前已映射的招式效果会贯通领域事件、观战投影、播放事件和中文信息。

## 明确未支持或部分支持

### 高优先级规则缺口

- 易失状态：混乱、束缚、蓄力、半无敌、可重复多回合攻击等，以及它们的行动前/回合末时机。
- 特殊伤害：反击、镜面反射、念波、一击必杀及依赖历史伤害的规则。
- 招式机制：太阳光束蓄力、梦话、梦游、优先度例外和其他未映射变化/伤害后效果。
- 场地状态：撒菱、反射壁、光墙及其他场地、己方场地或屏障效果。

### 资料与特性缺口

- 大量 PokeAPI `effect_id` 仍未映射；当前策略是避免把它们静默当作普通伤害或无效果变化招式。
- 未建立“所有效果编号 -> 支持状态/原因”的自动导入报告。
- 大量 Gen3 特性仍未实现，包括接触触发、强制行动、变身、形态、复制、天气外倍率和复杂入场时机。
- 尚未建立由真实队伍和固定种子构成的长期回归战报集合。

## 当前推荐推进顺序

1. 建立招式效果映射报告，列出每个资料效果编号的支持状态，防止资料覆盖率不透明。
2. 引入通用易失状态所有权，优先完成太阳光束蓄力、混乱和束缚；不要为单个招式建立孤立字段。
3. 在该状态模型上完成多回合/半无敌招式、梦话/梦游和相关状态交互。
4. 增加场地状态模型并实现反射壁、光墙、撒菱等可组合效果。
5. 按资料出现频率和规则复杂度继续扩展特性，并为每个触发时机补端到端事件测试。
6. 用真实队伍和固定种子建立回归战报，锁定规则和事件顺序。

## 设计与验收约束

1. `battle-domain` 是战斗规则、可变战斗状态和确定性随机数的唯一所有者。
2. 每个影响战局的变化都必须成为领域事件，再由应用、会话和播放层投影；禁止只修改最终快照。
3. 领域层按效果模型解释招式，资料导入层负责把 PokeAPI 的效果编号映射为该模型；禁止按招式 ID
   字符串散落分支。
4. 每个随机判定的消耗顺序必须确定、可测试；未到达的后续效果不得消耗随机数。
5. 规则必须按行动前、命中、伤害后、行动后、回合末和入场时机明确结算。
6. 无副作用且无状态的 crate 必须维持生产代码行覆盖率 100%。当前门禁覆盖
   `punctum-grid`、`punctum-input`、`punctum-terminal`、`punctum-ui` 和 `game-asset-plan`；
   有战局状态的 `battle-domain` 不适用该硬指标，但每条新增规则必须有直接测试。
